# Submission Metadata Checklist

This folder is intended for the non-manuscript items typically required during journal submission.

## Recommended Files to Place Here

### Administrative metadata
- `title_page.docx`
- `author_affiliations.docx`
- `corresponding_author_info.txt`

### Journal-facing documents
- `cover_letter.docx`
- `highlights.docx` or `highlights.txt`
- `suggested_reviewers.docx` or `suggested_reviewers.xlsx`

### Required declarations
- `funding_statement.txt`
- `conflict_of_interest.txt`
- `data_availability_statement.txt`
- `code_availability_statement.txt`
- `CRediT_authorship_statement.txt`
- `acknowledgements.txt`

### Optional but useful
- `response_to_reviewers.docx` (for revision stages)
- `submission_notes.md`

## Practical Submission Logic

If you are asking “what else besides `/data`, `/code`, `/figure`?”, the short answer is:

- `manuscript`
- `references`
- `metadata`
- `supplementary`

Those are usually the missing pieces that make a package actually submission-ready.
